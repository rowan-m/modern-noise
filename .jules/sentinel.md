## 2026-03-22 - CSP connect-src and SHA-256 for inline scripts
**Vulnerability:** When implementing Content-Security-Policy (CSP) headers via `firebase.json` for security enhancements, restricting `connect-src` to just `'self'` or overly tight bounds breaks the core application integration with Firebase and Google APIs.
**Learning:** The Firebase SDK and related systems (like Google Analytics, Firestore, Auth) rely strictly on outbound API calls. Failing to explicitly whitelist `https://*.googleapis.com` and `https://*.firebaseio.com` leads to massive regressions masquerading as security. Furthermore, correctly calculating the SHA-256 hash for inline scripts requires precise inclusion of all whitespace exactly as the browser parses it.
**Prevention:** Always allow `https://*.googleapis.com https://*.firebaseio.com` inside the `connect-src` directive for Firebase projects to preserve external data communication securely. Always thoroughly verify CSP hashes via real browser parsing before committing.
